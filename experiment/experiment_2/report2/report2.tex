  \section{目的}
  近年，クラウド化の影響のによってネットワークの重要性がより大きくなった．しかし，``\verb|Amazon Web Service|''(以下:``\verb|AWS|'')や``\verb|Azure|''，``\verb|Google Cloud Platform|''などによってネットワーク内部に触れる機会は大きく減少している．それが引き起こす問題として，この実験直前に起こった``\verb|AWS|''のシステム障害を起因とする大規模なシステム障害である．これは市場が``\verb|AWS|''の寡占状態にあったことも問題の一つであるが，ネットワーク内部に触れるエンジニアの割合が市場の規模に適していないからであると私は考えている．そのため，有事の際にネットワークを構築可能になるためにデファクトスタンダードである``\verb|Cisco|''社製のスイッチやルータを用いてスイッチを用いて\verb|LAN|環境の構築やルータのルーティングテーブルの設定を行うことができる様になる必要がある．
  \section{内容}
  今回構築したネットワークは主にレイヤ3(以下:\verb|L3|)までの階層で動作する．いくつかの段階からこのネットワークは構築されている．
  \begin{enumerate}[(1). ]
	  \item デバイス同士で通信を行うためにL2のスイッチ(以下:スイッチ)の\verb|MAC|アドレステーブルを用いて，\verb|Ethernet|で通信を行う\cite{Cisco_LAN_switch_text}．ただし，昨今のほとんどでは\verb|IP|アドレスが用いられている．今回もその例外ではない．そのため，\verb|ARP|コマンドを用いて\verb|IP|アドレスから相手の\verb|MAC|アドレスを調べる．
    \item 今実験ではスイッチの数が足りないため，スイッチの\verb|VLAN|機能を用いる．この機能でスイッチを仮想的に2つに分割する．これにより，異なる\verb|VLAN|に接続したデバイス同士は通信が行えない．ユーザがポートを選択し，静的にポートをそれぞれの\verb|VLAN|に割り振る．
    \item その異なる\verb|VLAN|に接続しているデバイス同士の通信を行うためにL3のルータを用いる．ルータに存在するルーティング機能を用いて，2つのセグメントに別れたデバイスに情報を送信する．
    \item 更に上位のルータを用いて，自班のルータのルーティングテーブルに他の班の情報を書き込み，他の班との通信を行う．上位ルータと接続のためにポートを用いるため，スイッチのポート数が足りなくなるため，今実験では\verb|Switch Virtual Interface|(以下:\verb|SVI|)を用いる．この機能は\verb|VLAN|の機能にルータのルーティング機能を搭載したものである．この機能によりルータのスイッチポートを\verb|VLAN|として扱うことができる． 本大学のネットワーク経由でyahoo.co.jpに接続する。この接続における名前解決には、本大学のDNSサーバーが用いられる． 
  \end{enumerate}

  \section{考察}
  今実験でVLANを構築する際に発生した事案として，VLANを構築した際にブロードキャストストームを発生させようとPingコマンドを実行した．その際に差口を間違ってしまい，VLAN1のポートからVLAN2のポートにLANケーブルを指してしまった．
  すると，異なるVLAN同士のデバイスにパケットを送ることができた．これは考えれば当たり前のことなのだが，異なるVLANに対してLANケーブルを通じてパケットを送ることができるため，通信が行えるのだ．\\

  友人に「どうやってパケットは目的地に近づいて行くのか？」という質問に対して納得させられる答えを提示できなかったため，それを以下に示す．

  まず同じネットワーク内にいるときは，スイッチ等のもつMACアドレス情報やすでに登録されている情報からパケットを送信する．同じネットワーク内にいない場合はデフォルトゲートウェイ，つまりルータに届く．
  このとき，今実験でも使用したルーティングテーブルを使用する．そのルーティングテーブルに従って次のルータに送信される．そのルータがまだ目的地ではなかった場合は同じようにルーティングテーブルから次のルータに送信される．
  実際にパケットは近づいてるという証拠も無くただ，ルーティングテーブルに従ってルータによって送り出されている．
