\section{プロジェクト: MIDIデバイスのハッキング}

\begin{frame}{なぜMIDIコントローラーなのか？}
  \begin{columns}
    \begin{column}{0.5\textwidth}
      \textbf{ストリームデック化への最適解}

      \vspace{1em}
      テンキーなどの標準HIDデバイスではなく、あえて中古のMIDI機材(AKAI MPD18等)を採用した理由。

      \vspace{1em}
      \begin{itemize}
        \item \textbf{誤入力の防止:} 通常の文字入力と信号が完全に独立している（エディタに「1」と入力されない）。
        \item \textbf{アナログ制御:} フェーダーを用いた0〜127の連続値（CC）による直感的なシステム音量操作。
        \item \textbf{Linux親和性:} Fedora等においてALSAドライバで即座に認識可能。
      \end{itemize}
    \end{column}
    \begin{column}{0.5\textwidth}

\begin{frame}[fragile]{システムアーキテクチャと実装}
  \textbf{Go言語によるクロスプラットフォーム・バックエンド}
  
  Windows/Linux両対応の常駐デーモンをGo言語で開発し、OSのAPIを直接叩いて入力を横取りする設計。

  \vspace{1em}
  \begin{lstlisting}[language=Go, caption=MIDI信号の傍受とOSコマンド実行のコアロジック]
// NoteOn（パッド押下）信号の検知
if msg.GetNoteOn(&channel, &key, &velocity) {
    fmt.Printf("ノート番号: %d が押されました\n", key)
    
    // 特定のノート番号でアクションを分岐
    if key == 36 {
        // OSのコマンドを直接キックする (例: OBSの操作等)
        exec.Command("xdg-open", "https://github.com").Start()
    }
}
  \end{lstlisting}
\end{frame}
